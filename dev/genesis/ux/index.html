Overview

The following items represent different widgets and elements that can be used in the design of a Genesis plugin. These elements are parsed by an XML stylesheet interpreter that will then render them into HTML.

Elements can also be created by calling them on genesis.core.Application.inflate like so:

{% highlight python %} ui = self.app.inflate('hosts:main') t = ui.find('list') for h in self.hosts: t.append(UI.DTR( UI.Label(text=h.ip), UI.Label(text=h.name), UI.Label(text=h.aliases), UI.HContainer( UI.TipIcon( iconfont='gen-pencil-2', id='edit/' + str(self.hosts.index(h)), text='Edit' ), UI.TipIcon( iconfont='gen-cancel-circle', id='del/' + str(self.hosts.index(h)), text='Delete', warning='Remove %s from hosts'%h.ip ) ), )) {% endhighlight %}

First, a container is found in the inflated XML (variable t above), then the elements are “appended” to the container by calling their name on UI like so.

Attributes of elements can be changed in the backend by calling genesis.ui.Element.set() like this:

{% highlight python %} ui.find('aliases').set('value', h.aliases) {% endhighlight %}

Complex Controls

Tabs

Shows a row of tabs with a space underneath for element pagination.

{% highlight xml %} ... ... {% endhighlight %}

Tabs are switched immediately, without generating events. But, a TabHeader with the **live=True** attribute will generate **tab/click** event. With **live=True**, you can also use **form** and **action** attributes. Applying the **live** attribute to TabControl will force all child TabHeaders to generate events.

Tree

A tree is a complex expandable widget. Can only be called from runtime. Example:

{% highlight python %} def on_session_start(self): self._tree = genesis.ui.TreeManager() def get_ui(self): ... tree = UI.TreeContainer(text='Root', id='/') sub = UI.TreeContainer(text='Subtree', id='/sub') node0 = UI.TreeContainerNode( UI.Label(text='node 0') ) node1 = UI.TreeContainerNode( UI.Label(text='node 1'), active=True ) tree.append(sub) sub.append(node0) tree.append(node1) self._tree.apply(root) # restore node states @event('treecontainer/click') def on_tclick(self, event, params, vars=None): self._tree.node_click('/'.join(params)) # save node state return '' {% endhighlight %}

Forms

FormBox

Yields the form/submit event on submission. The FormBox is displayed as a set of fields and an OK + Cancel buttons. The optional miscbtn attribute allows a third button to be added to the form.

{% highlight xml %} ... ... {% endhighlight %}

SimpleForm

Same as FormBox, but doesn’t have any buttons or padding. Yields the dialog/submit event on submission.

{% highlight xml %} {% endhighlight %}

DialogBox

Yields the dialog/submit event on submission. Same as a FormBox, but is displayed as a modal.

InputBox

A DialogBox with a single text input. Yields the dialog/submit event on submission. Input can be a TextInput, TextInputArea or CodeInputArea depending on the extra attribute (empty, ‘area’ or ‘code’ values).
